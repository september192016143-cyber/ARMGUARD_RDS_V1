Excel Template — Required & Optional Columns

The first row must be a header. Column names are case-insensitive and spaces are treated as underscores (e.g. SERIAL NUMBER and serial_number are both accepted). Extra columns are ignored. A single sheet can contain a mix of pistol and rifle rows (when no Batch Model is selected above).

Column Required Notes
item_typeYes*pistol or rifle. Optional when Batch Model is selected.
modelYes*Must match an exact model string (see below). Optional when Batch Model is selected.
serial_numberYesMust be globally unique
item_numberYesUnique per model. Numeric values are zero-padded to 4 digits (e.g. 1 → 0001)
property_numberNoGovernment property number. Must be globally unique if provided.
factory_qrRifle onlyRequired when model = M4 Carbine DSAR-15 5.56mm
item_conditionNoDefault: Serviceable. Options: Serviceable, Unserviceable, Lost, Tampered
item_statusNoDefault: Available. Options: Available, Issued, Under Maintenance, Decommissioned
descriptionNoFree-text notes

{# ── Rules ──────────────────────────────────────── #}
Import Rules
  • Each row is processed independently — a bad row is skipped with a warning and does not affect others.
  • Duplicate serial numbers, item numbers (per model), or property numbers cause the row to be skipped.
  • QR code and Item Tag images are generated automatically for every successfully imported record.
  • The Issued status cannot be set via import — use a Transaction instead.
  • Blank rows in the middle of the sheet are silently ignored.
